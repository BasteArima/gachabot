# Контент-инструменты (темы)

Контент бота (редкости, сеты, карты) описывается одним декларативным файлом
`theme.json`. Карты ссылаются на редкости и сеты **по имени** — никаких магических
id. Это единый источник правды: можно версионировать в git, ревьюить, подменять.

## Формат `theme.json`

```jsonc
{
  "rarities": [
    { "name": "common", "drop_chance": 60, "base_reward": 10,
      "pity_threshold": 0, "craft_cost": 5,
      "requires_fragments": false, "fragments_required": 0 }
  ],
  "sets": [
    { "name": "Базовый сет", "buff_type": "power_percent",
      "buff_value": 10, "reward_points": 500 }
  ],
  "cards": [
    { "name": "Карта 1", "rarity": "common", "power_level": 10,
      "image_url": "https://.../card1.webp", "set": "Базовый сет" }
  ]
}
```

`set` у карты опционален. Пример — `theme.example.json` в корне.

## Самый простой путь — через Телеграм (только админ)

Бот понимает админ-команды (отвечает только пользователю с `ADMIN_TELEGRAM_ID`):

- **`/theme_export`** — бот пришлёт текущую тему файлом `theme.json`.
  Скачай его и открой в редакторе (или сразу правь руками).
- **`/theme_import`** — применить тему: прикрепи `theme.json` к сообщению, а в
  подписи к файлу укажи `/theme_import`. Бот покажет предпросмотр (что изменится)
  и спросит подтверждение кнопкой — запись в БД только после «✅ Применить».
- **`/admin`** — список всех админ-команд.

Цикл: `/theme_export` → правка в редакторе → прикрепить файл с `/theme_import`.
Ничего коммитить и деплоить не нужно.

## CLI: `cmd/seed`

Использует те же `POSTGRES_*` из `.env`, что и бот.

```bash
# Проверить файл (без БД): ссылки, дубликаты, баланс дроп-шансов
go run ./cmd/seed validate theme.json

# Выгрузить текущую БД в файл (получить рабочую тему как старт)
go run ./cmd/seed export theme.json

# Применить к БД — АДДИТИВНО (upsert по имени, ничего не удаляет)
go run ./cmd/seed import theme.json
go run ./cmd/seed import theme.json --dry-run   # показать, что изменится, не трогая БД
```

**Безопасно для прода:** карты сопоставляются по имени, существующие сохраняют
свой `id`, поэтому коллекции игроков (`user_inventory`) не страдают. Ничего не
удаляется.

> `--reset` удаляет ВСЕ редкости/сеты/карты и каскадно — коллекции игроков.
> Только для пустой/дев-базы. Требует ввода `RESET` в консоли.

## Визуальный редактор

`tools/theme-editor/index.html` — статичная страница, без сервера и сборки.
Открой файл в браузере (или через любой статик-сервер):

```bash
python -m http.server 8910 --directory tools/theme-editor
# затем http://localhost:8910
```

- **Загрузить** — открыть существующий `theme.json`
- редактируешь редкости/сеты/карты полями; карты выбирают редкость/сет из
  выпадающих списков (рассинхрон невозможен)
- сверху — живая валидация и сумма дроп-шансов
- **Скачать theme.json** — сохранить результат

Типичный цикл: `seed export` → правка в редакторе → `seed import`.
